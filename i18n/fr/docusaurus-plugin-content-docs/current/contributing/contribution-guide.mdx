# Guide de contribution {#contribution-guide}

Si vous souhaitez contribuer au code source ou à la documentation de necord, n'hésitez pas à forker le dépôt et à soumettre une
pull request. Nous utilisons ESLint pour imposer un style de code cohérent, donc le configurer dans l'éditeur de votre choix
réduit les frictions lors de la contribution à ce projet.

## Configuration {#setup}

Vous aurez besoin de [Node.js](http://nodejs.org) **version 18+**, et de [npm](https://www.npmjs.com/).

Pour vous préparer à travailler sur le code source, veuillez procéder comme suit :

1. Forkez et clonez le dépôt. Assurez-vous d'être à jour avec la branche **master** !
2. Installez les dépendances en utilisant `yarn install` ou `npm install`
3. Codez à cœur joie !
4. Exécutez `yarn lint` pour lancer ESLint et vous assurer que les modifications respectent notre guide de style
5. [Soumettez une pull request](https://github.com/necordjs/necord/compare) (Assurez-vous de suivre le [format de commit conventionnel](https://github.com/SocketSomeone/necord/blob/main/.github/COMMIT_CONVENTION.md))

Un aperçu général des outils utilisés :

- [TypeScript](https://www.typescriptlang.org/) comme langage de développement
- [Eslint](https://eslint.org/) pour le style de code
- [Prettier](https://prettier.io/) pour le formatage du code

## Contribuer à la documentation {#contributing-to-documentation}

Si vous souhaitez contribuer à la documentation, vous pouvez le faire en modifiant les fichiers du dépôt [docs](https://github.com/necordjs/necord.org).
La documentation est rédigée en Markdown, vous pouvez donc utiliser n'importe quel éditeur Markdown pour apporter des modifications. Utilisez `yarn dev` pour démarrer un serveur local et prévisualiser vos modifications.

Avant de soumettre une pull request, veuillez vous assurer que :

- Utilisez [Markdown](https://www.markdownguide.org/) pour le formatage
- Utilisez [MDX](https://mdxjs.com/) pour intégrer des composants React
- Utilisez [GitHub Flavored Markdown](https://github.github.com/gfm/) pour les blocs de code
- Utilisez des liens d'ancrage pour les liens internes (par ex. `#contributing-to-documentation`)
